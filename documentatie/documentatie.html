1. Introducere

1.1 Scopul

Scopul acestui document este de a prezenta cerințele software pentru proiectul Pet Adoption. Este o platformă web care oferă posibilitatea de a adopta animale de companie.

Domeniul de aplicare include: crearea și administrarea profilelor animalelor oferite spre adopție, un motor de căutare după criterii (rasă, culoare, specie etc.) și un proces de solicitare adopție.

Nu sunt incluse în versiunea inițială: modul de chat în timp real și forumul comunității.

1.2 Convenții de Document

  • Tipografie:
    • Text îngroșat pentru elemente de interfață
    • Text înclinat pentru termeni noi sau referințe
  • Identificatori pentru cerințe: F-x.x pentru cerințe funcționale, iar cele nefuncționale sunt menționate în manieră directă.
  • Roluri definite:
    • Stăpânul: oferă un animal spre adopție
    • Utilizatorul: dorește să adopte
    • Vizitatorul: acces limitat, neautentificat
    • Admin: control total asupra datelor

1.3 Audiența Țintă și Sugestii de Citire

Documentul se adresează următoarelor categorii:

  • Evaluatorul: evaluează specificațiile
  • Dezvoltatorul: ghid pentru implementare
  • Testerul: elaborare teste de validare

Recomandări:

  • Evaluator – document complet
  • Dezvoltator și Tester – secțiunile 3, 4, 5

Actualele metode de adopție, precum social media sau anunțuri generale, nu sunt sigure. Platforma propusă va centraliza și securiza procesul de adopție pentru toate rolurile implicate.

2. Descriere Generală

2.1 Perspectiva Proiectului

Site-ul de Pet Adoption este un produs nou, de sine stătător, conceput și dezvoltat de la zero. Nu este o versiune îmbunătățită sau o înlocuire a unui alt sistem existent. Este o alternativă mai sigură și specializată la platformele generale de anunțuri sau grupurile sociale pentru adopții de animale.

Proiectul oferă un spațiu digital exclusiv dedicat interacțiunii dintre stăpân și potențialul adoptator, într-un mediu sigur și controlat.

Diagrama de context (C4 - nivel 1):

2.2 Funcțiile Proiectului

  • Managementul Conturilor:
    • Crearea unui cont nou
    • Autentificare utilizatori existenți
  • Funcționalități pentru Stăpân:
    • Creare și publicare anunț de adopție
    • Vizualizare și gestionare aplicații
  • Funcționalități pentru Utilizator:
    • Căutare și filtrare anunțuri
    • Vizualizare animale
    • Aplicare prin formular
  • Funcționalități pentru Admin:
    • Administrare conturi
    • Administrare anunțuri

2.3 Clasele și Caracteristicile Utilizatorului

Proiectul identifică patru clase principale de utilizatori, fiecare cu caracteristici și nevoi diferite:

  • Stăpân: utilizator autenticat, nivel tehnic mediu, responsabil cu publicarea anunțurilor și gestionarea cererilor de adopție.
  • Utilizator: doritor de adopție, nivel tehnic redus-medie, caută, filtrează și aplică pentru adopții.
  • Admin: super-utilizator cu acces deplin, nivel tehnic ridicat, gestionează întregul sistem.
  • Vizitator: neautentificat, nivel tehnic scăzut, are acces doar la vizualizarea anunțurilor.

2.4 Mediul de Operare

Server: dezvoltare locală cu XAMPP, incluzând Apache, PHP și MySQL. Permite testare controlată și replicare a mediului de producție.

Client: browser web modern (Chrome, Firefox, Safari, Edge). Aplicația este responsive și optimizată pentru toate dispozitivele.

2.5 Constrângerile de Implementare și Design

  • Back-End: PHP
  • Front-End: HTML5, CSS, fără framework-uri complexe
  • Bază de date: MySQL
  • Server: Apache

Design: responsive, conform ghidului de stil (referința 2).

Limba: exclusiv engleză

Securitate: parolele vor fi stocate cu password_hash(), iar datele validate și sanitizate pentru a preveni SQLi și XSS.

Control versiuni: Git

2.6 Documentația Utilizatorului

Ghidaj contextual în interfață: texte explicative, etichete intuitive, mesaje de eroare clare.

2.7 Presupuneri și Dependențe

Presupuneri:

  • Utilizatorii oferă informații corecte și au competențe digitale de bază
  • Accesul la internet și suport pentru HTML5/CSS

Dependențe:

  • Apache, PHP, MySQL, XAMPP
  • Furnizor de hosting și sistem DNS funcțional

3. Cerințele Interfeței Externe

3.1 Interfețele Utilizatorului

Aplicația oferă o interfață web intuitivă, modernă și adaptabilă, proiectată pentru ușurință în utilizare atât pentru utilizatori cât și pentru administratori.

Principii Generale:

  • Consistență Vizuală: paletă de culori unitară (#9990DA), fonturi moderne (Arial, Segoe UI), design coerent.
  • Layout Standard: Header, Conținut Principal, Footer.
  • Responsive Design: Complet responsiv, rearanjare sub 768px.

Componente Standard:

  • Header: Logo, navigare (Home, Adopt, Rehome a Pet, About), butoane în funcție de sesiune (Login/Register, Profil, Logout, Admin Panel).
  • Sidebar / Filtre: Checkboxes, dropdown-uri pentru căutare detaliată.
  • Carduri animale: Imagine, detalii, buton „More Info”.
  • Formulare: Adăugare animal, cerere adopție, login, editare profil.
  • Footer: Linkuri utile și informații legale.

Exemple de Ecrane:

  • Pagina Căutare: searchMenu.php
  • Pagina Detalii Animal: petPage.php?id=...
  • Panou Admin: admin/animal_pages_scripts/admin_pet_requests.php

3.2 Interfețele Hardware

Aplicația este web-based și nu necesită hardware special. Funcționează pe:

  • Client: orice browser modern (Chrome, Firefox, Safari, Edge)
  • Server: PHP + MySQL (Apache/Nginx)

Nu există interacțiuni cu hardware specializat (camere, senzori etc.).

3.3 Interfețele Software

Aplicația interacționează cu următoarele componente:

  • Server Web: Apache/Nginx
  • PHP: rularea aplicației (recomandat PHP 8+)
  • MySQL/MariaDB: baza de date
  • Librării PHP: mysqli, password_hash(), password_verify()
  • CSS extern: fișiere CSS pe pagini (ex: design/petPage.css)
  • Feed RSS: generare fișier XML cu rss.php

3.4 Interfețele de Comunicare

Comunicarea se face prin protocoale standard:

  • HTTP/HTTPS: pentru comunicarea client-server
  • POST/GET: metode pentru trimiterea datelor
  • Email (opțional): notificări de aprobare / respingere cerere
  • Notificări interne: salvate în baza de date și afișate utilizatorului

Aplicația este compatibilă cu rețele LAN/WAN și internet public. Nu depinde de protocoale industriale.

4. Caracteristici de Sistem

Acest sistem oferă funcționalități pentru a facilita adopția animalelor printr-o platformă web, unde utilizatorii pot căuta animale, trimite cereri de adopție, adăuga animale spre adopție și gestiona profilul personal. Mai jos sunt descrise cele mai relevante caracteristici funcționale.

4.1 Funcționalitate: Căutare și Filtrare a Animalelor

Descriere și Prioritate: Permite utilizatorilor să vizualizeze animale disponibile și să aplice filtre. Prioritate: Înaltă

Stimulus / Răspuns:

  • Stimulus: accesare pagină de căutare → Răspuns: afișare listă animale și filtru
  • Stimulus: selectare criterii → Răspuns: actualizare listă rezultate

Cerințe Funcționale:

  • [F-1.1] Afișare listă animale disponibile
  • [F-1.2] Filtrare după tip, rasă, culoare, vârstă, mărime, sex, țară, județ
  • [F-1.3] Actualizare imediată după aplicare filtre
  • [F-1.4] Redirecționare la pagina de detalii

4.2 Funcționalitate: Trimitere Cerere de Adopție

Descriere și Prioritate: Permite trimiterea de cereri de adopție. Prioritate: Înaltă

Stimulus / Răspuns:

  • Stimulus: clic pe „Adoptă” → Răspuns: afișare formular
  • Stimulus: trimitere formular → Răspuns: salvare cerere, notificare admin

Cerințe Funcționale:

  • [F-2.1] Doar utilizatori autentificați pot trimite cereri
  • [F-2.2] Cererea salvată cu status „pending”
  • [F-2.3] Notificare pentru admin
  • [F-2.4] Mesaj de confirmare pentru utilizator

4.3 Funcționalitate: Adăugare Animal spre Adopție

Descriere și Prioritate: Permite adăugarea de animale. Prioritate: Medie

Stimulus / Răspuns:

  • Stimulus: accesare formular → Răspuns: afișare formular complet
  • Stimulus: completare și trimitere → Răspuns: salvare cerere și notificare admin

Cerințe Funcționale:

  • [F-3.1] Formular cu detalii generale, sănătate, imagine, vaccinuri, calendar alimentar
  • [F-3.2] Validare și salvare în pet_requests
  • [F-3.3] Notificare către admin

4.4 Funcționalitate: Administrare Cereri

Descriere și Prioritate: Adminii gestionează cereri. Prioritate: Înaltă

Stimulus / Răspuns:

  • Stimulus: accesare panou notificări → Răspuns: afișare cereri
  • Stimulus: clic pe „Approve” / „Deny” → Răspuns: actualizare status, notificare utilizator

Cerințe Funcționale:

  • [F-4.1] Vizualizare cereri în așteptare
  • [F-4.2] Aprobare / respingere cerere
  • [F-4.3] Notificare utilizator

4.5 Funcționalitate: Autentificare și Autorizare

Descriere și Prioritate: Autentificare utilizatori și control acces. Prioritate: Înaltă

Stimulus / Răspuns:

  • Stimulus: introducere username și parolă → Răspuns: validare și creare sesiune

Cerințe Funcționale:

  • [F-5.1] Autentificare și înregistrare
  • [F-5.2] Verificare parolă cu password_verify
  • [F-5.3] Acces doar la conținut propriu
  • [F-5.4] Acces admin la panou de control

5. Alte Cerințe Ne-Funcționale

5.1 Cerințe de Performanță

  • Timp de răspuns maxim: 2 secunde pentru 95% dintre cereri.
  • Încărcare completă a paginilor: sub 3 secunde (conexiune 100Mbps).
  • Baza de date: minim 100 conexiuni simultane fără impact negativ.
  • Sistemul suportă 500 cereri/minut fără degradare vizibilă.

Raționament: Performanța ridicată este esențială pentru o experiență fluentă și scalabilitate viitoare.

5.2 Cerințe de Siguranță

  • Blocarea fișierelor periculoase în upload (ex: .exe, .js).
  • Mesaj prietenos în caz de eșec al bazei de date, fără informații sensibile.
  • Validare și igienizare completă a datelor introduse de utilizator.
  • Confirmare suplimentară pentru acțiuni critice.
  • Respectarea OWASP și standardelor GDPR/UE relevante.

5.3 Cerințe de Securitate

  • Autentificare: utilizator + parolă (hashuită cu bcrypt).
  • Protejarea paginilor sensibile prin autentificare și verificare permisiuni.
  • Parolele nu se transmit sau stochează în clar.
  • Protecție date sensibile la nivel de administrare.
  • Apărare împotriva: CSRF, XSS, SQLi.
  • Accesul la conturile admin este restricționat și auditat.
  • Recomandare: conformitate cu ISO/IEC 27001.

5.4 Atributele de Calitate ale Software-ului

  • Disponibilitate: minim 99% uptime.
  • Usability: interfață clară, intuitivă pentru utilizatori non-tehnici.
  • Flexibilitate: arhitectură extensibilă fără modificări majore.
  • Portabilitate: compatibil cu PHP 8+ și MySQL 5.7+.
  • Testabilitate: cod modular, ușor de testat unitar și integrat.
  • Interoperabilitate: export RSS, CSV, JSON.

5.5 Reguli de Business

  • Numai utilizatorii autentificați pot posta sau aplica la adopție.
  • Adminii pot aproba/respinge anunțuri de adopție.
  • Fiecare utilizator poate vizualiza doar cererile proprii.
  • Un animal poate fi marcat o singură dată ca „adoptat”.
  • O cerere de adopție este unică per utilizator/animal.
  • Adminii primesc notificări la cereri noi de adopție.
